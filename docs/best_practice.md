# 场景案例

## 我希望将金数据的表单，集成到我的网页、移动端网页、微信公众号、App、小程序中。

对于网页嵌入，无需额外开发，金数据表单已经提供了 iframe 和 script 两种嵌入方式。

对于公众号，只需要将你的表单对外地址 URL 配置到相应菜单或者相应功能即可。

对于 App，请使用 Webview 的方式直接嵌入金数据表单即可。

对于小程序，由于微信小程序有域名限制，所以请联系你的客户成功经理协助配置。

## 我希望金数据表单收集的数据，都可以回传到我自己的服务器上，用作系统集成或者数据分析。

> 推荐功能：Webhook

可以在金数据表单上开启 Webhook 功能，配置自定义的服务器 URL。金数据系统可以将所有收集到的新增数据全部以 JSON 格式推送到指定的服务器上。

金数据表单 Webhook 支持「新增数据」和「变更数据」两种触发事件。所以即使数据发生变更，也可以同步最新的结果到你的服务器。

对于企业版套餐用户，可以设置企业全局 Webhook。配置了全局 Webhook 后，企业内所有的表单都可以推送到指定的服务器上。

Webhook 在推送时，会包含表单的标识符，所以你的系统可以通过标识符来判断是哪一个表单。

## 我希望通过不同的渠道来分发我的表单，在回收的数据上可以判断是哪个渠道。

使用金数据表单扩展属性功能即可实现。

## 我希望通过我的 App、网页，将已经识别的一些信息传递给金数据表单（例如用户ID等）。

对于简单的单个信息，可以使用金数据表单的扩展属性字段。

对于复杂的信息传递，例如：

1. 我需要传递多个信息，例如用户ID、用户等级、渠道；
2. 我需要确保这些信息是真实有效，避免恶意用户篡改；

可以使用金数据表单的 URL 传参功能即可实现。

## 我希望将金数据表单数据和我的其他系统完全集成互通。

可以使用金数据的 API（v1）。

金数据的 API（v1）提供了表单的查询，以及表单数据的增加、查询、修改、删除能力。
