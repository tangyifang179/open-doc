# 场景案例

如果你不确定该使用 API、Webhook、URL 传参、表单嵌入还是 MCP，可以先从下面的场景开始选择。

## 让 AI 助手操作金数据

适合希望通过自然语言查询表单、整理数据、创建表单或处理日常数据工作的团队。

推荐能力：[MCP Server](/mcp/)

你可以实现：

- 让 AI 助手查询某个表单的数据
- 让 AI 根据描述创建报名表、反馈表、申请表
- 让 AI 汇总表单数据并生成运营统计
- 在 AI Agent 工作流中接入金数据

下一步：查看 [MCP Server](/mcp/)。

## 将表单嵌入网站、App 或小程序

适合希望在自己的产品、官网、活动页、公众号、App 或小程序中收集数据的团队。

推荐能力：[表单嵌入](/embedded/overview/)

你可以实现：

- 在官网或活动页中嵌入报名表
- 在移动端网页或 App WebView 中收集用户信息
- 在公众号菜单或推文中引导用户填写表单
- 在小程序中接入金数据表单

下一步：查看 [表单嵌入](/embedded/overview/)。

## 表单提交后同步到自己的服务器

适合希望在用户提交报名、申请、反馈、线索或工单后，实时触发外部系统处理的团队。

推荐能力：[Webhook](/webhook/)

你可以实现：

- 用户提交表单后自动同步到 CRM 或工单系统
- 新报名产生后自动发送通知
- 将表单数据实时写入内部数据库或数据管道
- 表单数据变更后同步更新外部系统记录

下一步：查看 [Webhook](/webhook/)。

## 记录渠道来源或外部用户信息

适合需要追踪不同渠道效果，或需要把外部系统中的用户 ID、订单 ID、客户 ID 传入表单的团队。

推荐能力：[URL 传参](/url_params/)

你可以实现：

- 判断每条表单数据来自哪个渠道、活动或广告
- 将外部系统中的用户 ID、订单 ID、客户 ID 写入表单数据
- 将用户填写内容和业务系统记录关联起来
- 对关键参数进行签名校验，避免被篡改

下一步：查看 [URL 传参](/url_params/)。

## 通过程序读取和管理表单数据

适合需要将金数据深度接入 CRM、工单系统、运营后台、数据仓库或内部业务系统的开发团队。

推荐能力：[API v1](/api_v1/)

你可以实现：

- 读取表单列表、表单结构和提交数据
- 创建、更新或删除表单数据
- 将表单数据定时同步到内部系统
- 构建基于金数据的后台工具或自动化流程

下一步：查看 [API v1](/api_v1/)。

## 如何选择

| 你的目标 | 推荐能力 |
|---|---|
| 让 AI 助手通过自然语言操作金数据 | [MCP Server](/mcp/) |
| 把表单放进自己的网站、App 或小程序 | [表单嵌入](/embedded/overview/) |
| 表单提交后，实时推送数据到服务器 | [Webhook](/webhook/) |
| 在表单链接中携带渠道、用户 ID、订单 ID | [URL 传参](/url_params/) |
| 通过程序读取、创建、修改或同步数据 | [API v1](/api_v1/) |
