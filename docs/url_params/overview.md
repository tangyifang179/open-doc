# 概览

金数据 URL 传参可以让你在表单链接中携带外部参数，并在用户提交表单后将这些参数保存到表单数据中。你可以用它记录渠道来源、用户身份、活动信息或业务系统中的关联 ID。

## 适合什么场景

URL 传参适合在用户打开或提交表单时，需要把外部系统中的信息一起带入金数据的场景。

你可以用 URL 传参：

- 记录不同渠道、广告、活动或推广链接带来的表单提交
- 将业务系统中的用户 ID、订单 ID、客户 ID 等信息传入表单
- 在表单提交后，把用户填写内容和外部系统记录关联起来
- 对关键参数做签名校验，避免用户手动篡改传参内容

## 快速开始

如果你是第一次使用 URL 传参，建议按下面的方式选择：

1. 只需要传入 1 个简单参数：查看 [扩展属性](/url_params/x_field/)
2. 需要把多个参数写入表单字段：查看 [表单字段 URL 传参](/url_params/form_field_url_params/)
3. 需要把多个参数写入账号全局字段：查看 [全局字段 URL 传参](/url_params/global_field_url_params/)

> 如果参数会影响用户身份、订单归属、渠道结算等重要业务逻辑，建议使用支持签名校验的传参方式，避免参数被篡改。

## 扩展属性

金数据表单扩展属性功能，支持在发布出去的表单中传入 1 个 URL 参数。最终填表人在提交表单时，这个参数会被保存到表单数据上。

发布表单的 URL 格式：

```text
https://jinshuju.net/f/TOKEN?x_field_1=$VALUE$
```

查看 [扩展属性详细说明](/url_params/x_field/)。

## 表单字段传参

金数据表单字段传参功能，支持在发布出去的表单中传入多个 URL 参数。最终填表人在提交表单时，这些参数会回填到表单的对应字段上。

表单字段传参支持对 URL 参数加签名校验，用于防止填表人篡改。

发布表单的 URL 格式：

```text
https://jinshuju.net/f/TOKEN?field_1=$VALUE$&field_2=$VALUE$&sign=$SIGNATURE$
```

查看 [表单字段传参详细说明](/url_params/form_field_url_params/)。

## 全局字段传参

金数据全局字段传参功能，支持在发布出去的表单中传入多个 URL 参数。最终填表人在提交表单时，这些参数会回填到账号的全局字段上，并显示在表单的数据页面。

全局字段传参支持对 URL 参数加签名校验，用于防止填表人篡改。

发布表单的 URL 格式：

```text
https://jinshuju.net/f/TOKEN?cusd=$JWT_ENCODED_DATA$
```

查看 [全局字段传参详细说明](/url_params/global_field_url_params/)。

## 常见问题

### URL 中含特殊字符导致无法正常传入参数

如果传入的参数中包含特殊字符，需要先进行 URL 编码。

| 特殊字符 | 在 URL 中的含义 | 转义字符 |
|---|---|---|
| `+` | URL 中 `+` 号表示空格 | `%2B` |
| `/` | 分隔目录和子目录 | `%2F` |
| `=` | 指定参数的值 | `%3D` |
| `?` | 分隔实际 URL 和参数 | `%3F` |
| 空格（space） | URL 中的空格可以用 `+` 号或者编码 | `%20` |
| `&` | 指定参数间的分隔符 | `%26` |
| `%` | 指定特殊字符 | `%25` |
| `#` | 表示书签 | `%23` |
