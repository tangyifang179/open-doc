# 概览

金数据 Webhook 可以在表单数据新增或修改时，将数据以 JSON 格式主动推送到你指定的服务器。你可以用它把表单提交、线索、报名、工单、反馈等数据实时接入业务系统或自动化流程。

## 适合什么场景

Webhook 适合需要在表单数据发生变化后，实时触发外部系统处理的场景。

你可以用 Webhook：

- 在用户提交表单后，自动同步数据到 CRM、工单系统或运营后台
- 在报名、申请、反馈等数据产生后，实时触发消息通知或后续流程
- 将金数据表单数据接入数据管道、自动化平台或内部服务
- 在表单数据被修改后，同步更新外部系统中的记录

## 快速开始

如果你是第一次接入 Webhook，建议按下面的顺序开始：

1. 查看 [配置 Webhook](/webhook/configuration/)，在表单设置中开启数据推送
2. 查看 [Webhook 数据格式](/webhook/data_format/)，了解金数据推送到服务器的 JSON 结构
3. 查看 [如何测试 Webhook](/webhook/how_to_try_webhook/)，确认你的服务器能正确接收数据
4. 查看 [Outbound IP List](/webhook/outbound_ip_list/)，如有需要，将金数据推送服务器 IP 加入白名单

> Webhook 需要你的服务器在收到推送后返回 2XX 状态码。上线前建议先用测试数据确认接收地址、数据格式和业务处理逻辑。

## Webhook 类型

### 1. 单表数据推送

你可以在表单的「设置 —— 数据推送」页面找到「将数据以 JSON 格式发送给第三方」，勾选开启后点击「字段对照表与 JSON 样例」展开，查看当前表单的字段对照表。

配置成功后，当前表单收到新数据或编辑数据时，金数据会将该数据通过 POST 推送到指定 URL。

### 2. 企业全局数据推送

企业版用户可以在企业设置中配置全局 Webhook。配置成功后，企业内所有子账号的所有表单都会生效。
