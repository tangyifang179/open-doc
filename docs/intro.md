# 金数据开放平台

金数据开放平台帮助开发者通过 API、Webhook、MCP 和表单嵌入，将表单数据连接到业务系统、自动化流程和 AI 工具。

## 选择接入方式

根据你要完成的任务，选择合适的接入方式：

- 让 AI 助手管理表单、查询和操作数据：查看 [MCP Server](/mcp/)
- 通过程序读取、创建、修改表单和数据：查看 [API v1](/api_v1/)
- 在表单新增或修改数据后推送到你的服务器：查看 [Webhook](/webhook/home/)
- 在表单链接中传入渠道、用户 ID 等外部参数：查看 [URL 传参](/url_params/overview/)
- 将金数据表单嵌入网页、公众号、App 或小程序：查看 [表单嵌入文档](/embedded/overview/)

如果你不确定该使用哪种能力，可以先查看 [场景案例](/best_practice/)。

## 开放能力

### MCP Server

金数据 MCP Server 允许 Claude、Cursor、Windsurf 等支持 MCP 的 AI 工具连接金数据。配置后，AI 助手可以通过自然语言帮助你管理表单、查询数据、操作数据和查看账户信息。

适合场景：

- 让 AI Agent 查询或整理表单数据
- 通过自然语言创建和管理表单
- 将金数据接入 AI 自动化工作流

### API v1

金数据 API v1 提供基于 HTTP 的接口能力，支持读取表单结构、查询表单数据、创建数据、修改数据和删除数据。

适合场景：

- 将金数据表单数据同步到内部系统
- 在业务系统中创建或更新金数据表单数据
- 构建 CRM、工单、运营后台、数据分析等集成

### Webhook

Webhook 可以在表单数据新增或变更时，将数据主动推送到你的服务器。

适合场景：

- 表单提交后自动触发业务流程
- 将线索、报名、工单、反馈实时同步到内部系统
- 与消息通知、自动化工具或数据管道集成

### URL 传参

URL 传参可以在表单链接中携带渠道、用户 ID、活动来源等外部信息，并写入最终提交数据。

适合场景：

- 追踪不同渠道的表单提交效果
- 将外部系统中的用户信息传入表单
- 在活动、投放、会员系统中识别来源

### 表单嵌入

你可以将金数据表单嵌入到网页、移动端页面、公众号、App 或小程序中，用于收集业务数据。

适合场景：

- 在官网或活动页中嵌入报名表
- 在 App 或 WebView 中收集用户信息
- 将表单作为业务系统中的数据录入入口
